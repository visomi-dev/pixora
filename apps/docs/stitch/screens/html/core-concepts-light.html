Core Concepts

Understand the fundamental architecture of Pixora and how it handles modern UI state.

terminal 1. The App Instance

The PixoraApp is the root of your application. It manages the global state registry, event bus, and the rendering lifecycle. Unlike traditional frameworks, Pixora uses a "headless-first" approach where the app instance can exist independently of the DOM.

main.px 
import { createPixoraApp } from '@pixora/core';

// Initialize the core engine
const app = createPixoraApp({
  root: '#app',
  theme: 'light',
  rendering: 'gpu-accelerated'
});

app.mount();

dashboard_customize 2. Layout Logic

Pixora implements a proprietary constraint-based layout engine. It calculates positions using a multi-pass system that ensures sub-pixel precision across all device types.

Flex-Grid Hybrid

Combines the flexibility of CSS Flexbox with the structural integrity of native canvas grids.

Auto-Reflow

Instantly recalculates layout trees without triggering expensive browser repaints.

extension 3. Atomic Components

Components in Pixora are atomic. They are designed to be immutable by default, which allows the framework to optimize rendering using a technique we call "Component Diffing".

info

Note: All components must extend the BaseComponent class to ensure they register correctly with the lifecycle manager.
